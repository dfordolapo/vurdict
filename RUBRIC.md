# Vurdict Portfolio Evaluation Rubric

This rubric defines the scoring criteria for the Vurdict AI platform. Each category is scored on a scale of 1-10.

---

## 1. Problem Framing
*Does the portfolio clearly explain the problem being solved?*

| Score | Description |
| :--- | :--- |
| **1/10** | No mention of a problem. Just a gallery of images or a list of features without context. |
| **5/10** | Mentions a general problem (e.g., "users needed a better way to X") but lacks depth, research data, or business context. |
| **10/10** | Clearly articulates a specific user pain point, business goal, and technical constraints. Includes a "How Might We" statement or a clear hypothesis. |

---

## 2. Process Visibility
*Does the portfolio show how decisions were made?*

| Score | Description |
| :--- | :--- |
| **1/10** | "The Big Reveal" approach. Only shows the final high-fidelity designs. Zero evidence of sketches, wireframes, or testing. |
| **5/10** | Shows some wireframes or a user flow, but doesn't explain *why* changes were made. The process looks linear and performative rather than messy and iterative. |
| **10/10** | Transparently shows the "middle" of the project: failed prototypes, user testing insights that changed the direction, and trade-offs made due to constraints. |

---

## 3. Outcome & Impact
*Does the work demonstrate measurable or observable results?*

| Score | Description |
| :--- | :--- |
| **1/10** | No mention of what happened after the design was finished. No "Lessons Learned" or results. |
| **5/10** | Qualitative success mentioned (e.g., "Stakeholders were happy") or hypothetical impact, but no hard data or specific user feedback. |
| **10/10** | Provides specific metrics (e.g., +15% conversion, 2min time-on-task reduction) or clear qualitative success (e.g., specific quotes from user testing or app store reviews). |

---

## 4. Visual Quality
*Does the portfolio communicate professionalism and design competence?*

| Score | Description |
| :--- | :--- |
| **1/10** | Poor typography, inconsistent spacing, "vibrant" colors that clash, and lack of alignment. Looks like a template from 2010. |
| **5/10** | Clean and passable. Follows standard UI patterns (Material, HIG) but lacks personality or "wow" factor. Minor spacing/consistency issues. |
| **10/10** | Exceptional use of typography, color theory, and white space. High attention to detail (micro-interactions, grid systems). Feels premium and modern. |

---

## 5. Niche & Positioning
*Does the designer communicate a clear area of focus?*

| Score | Description |
| :--- | :--- |
| **1/10** | "I design anything." A confusing mix of logos, web design, 3D modeling, and UX without a clear narrative. |
| **5/10** | Identifies as a "Product Designer" but the work is very generic. No specific industry (Fintech, Health) or platform (SaaS, Mobile) specialty shown. |
| **10/10** | Strong positioning (e.g., "Senior Product Designer specializing in B2B SaaS complex workflows"). The portfolio work backs up this specific expertise. |

---

## 6. Trust & CTA Strength
*Does the portfolio establish credibility and provide a clear next step?*

| Score | Description |
| :--- | :--- |
| **1/10** | No "About" page, no contact info, and no way to hire the designer. Social links are broken or non-existent. |
| **5/10** | Standard "Hire Me" button and a basic LinkedIn link. Credits are mentioned but a bit vague. |
| **10/10** | Strong external validation (testimonials, awards, logos of past companies). Clear, prominent CTA (Call to Action) that aligns with the user's current goal. |
